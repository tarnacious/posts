vim-slime

Today I found the awesomeness that is vim-slime. Slime is the "The Superior Lisp Interaction Mode for Emacs", I can almost hear the emacs crowd laughing at me.

For those that use Vim and don't use slime or something similar, this is my understanding of what it's awesome.

Text can be sent from any process to the stdin of a screen or tmux session. The process in this case is Vim and the screen/tmux session is my terminal.

First some background on GNU Screen. Screen is really neat a terminal multiplexer (you can run multiple terminals. The multiplexed shell processes are child processes of the screen process, which is not a child of the terminal window. This means a screen process its child processes keep running if you close the terminal window. You can re-connect to it later, this is what makes vim-slime possible.

Here is an screen shot of this in use. On the left is me in gvim writing some aweful Clojure [1]. On the right is a screen buffer in which I started a Clojure REPL. When I want to try run some code send any text selection to with REPL in a keystroke (but I'm using the default key binding of ).

It doesn't have to be a Clojure REPL either, you can send anything to a screen shell. You could run git commands, find, grep, sed. Like with the Clojure REPL we can even interact with any terminal programs that use STDIN.

This concept can be taken even further. You can even connect to a tmux session over SSH and share a terminal or a terminal program like vim to do remote pairing!
